Algorithm Analysis

Asymptotic Notation

Big O (O)

Upper bound: \( f(n) = O(g(n)) \)

Example: \( 3n^2 + 2n + 1 = O(n^2) \)

Big Œ© (Œ©)

Lower bound: \( f(n) = \Omega(g(n)) \)

Example: \( n^2 + 3n = \Omega(n^2) \)

Big Œò (Œò)

Tight bound: \( f(n) = \Theta(g(n)) \)

Example: \( 2n^2 + n = \Theta(n^2) \)

Complexity Theory

P (Polynomial Time)

P

Problems solvable in polynomial time by a deterministic algorithm.

Sorting Shortest Path Linear Programming

NP (Nondeterministic Polynomial)

NP

Problems where solutions can be verified in polynomial time.

Traveling Salesman Boolean Satisfiability Graph Coloring

NP-Complete

NPC

Hardest problems in NP - if any is in P, then P = NP.

Hamiltonian Cycle Knapsack Problem Subset Sum

Classic Algorithms

Sorting Algorithms

O(n log n)
  • Quick Sort: Divide and conquer, average O(n log n)
  • Merge Sort: Stable sort, guaranteed O(n log n)
  • Heap Sort: In-place, O(n log n) worst case

Graph Algorithms

Varies
  • Dijkstra: Shortest path, O((V+E) log V)
  • Bellman-Ford: Negative edges, O(VE)
  • Floyd-Warshall: All pairs, O(V¬≥)

Dynamic Programming

Varies
  • Fibonacci: Memoization vs tabulation
  • Knapsack: 0/1 and unbounded variants
  • LCS: Longest common subsequence
